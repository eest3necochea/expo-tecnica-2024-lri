Control de versiones: GIT HUB

Durante el desarrollo de este proyecto incluimos el uso de Git Hub, una plataforma que permite subir archivos GIT. GIT, es un controlador de versiones que gestiona los cambios que se realizan sobre los elementos de algun producto o su cofiguración. La siguiente documentación incluye las habilidades adquiridad con GIT HUB:

  • Generación de Token: Cuenta de GitHub> Perfil> Setting> developer settings> Personal Acces token> Classic> Generar nuevo token> generar nombre> generar días (90 dias)> guardar URL.
  • Clonar el repositorio principal: git clone > enlace del repositorio.
  • Entramos a la carpeta:cd> expo-técnica-2024 (nombre del archivo).
  • Generar la rama grupo2: git branch grupo2 (nombre de la rama).
  • Entrar a la rama grupo2: git checkout grupo2.
  • Editar el archivo: Code .
  • Ver el estado del archivo: git status.
  • Cambiar el estado del archivo. Confirmar cambios: git add .
  • Comprobar el cambio del archivo: git status.
  • Ver contenido del archivo: git log.
  • Crear Commit: git commit -m “mensaje”.
  • Subir cambios al repositorio: git push origin grupo2 (nombre de la rama).

La Metodología Kanban

En nuestro proyecto, implementamos la metodología Kanban para gestionar tareas de manera eficiente.
Creamos un tablero dividido en columnas: "Por Hacer", "En Progreso" y "Completado". Cada tarea se representó con una tarjeta, lo que nos permitió visualizar el estado del trabajo.
Establecimos límites en el trabajo en progreso, restringiendo el número de tareas en "En Progreso". Esto mejoró nuestra concentración y redujo los tiempos de entrega.

PROYECTO RASPBERRY PI

- Comenzamos reformateando el router para dejarlo con su ip y contraseña de fábrica.

- Entramos a la red del router con un teléfono, empleando la contraseña escrita en la etiqueta.

- A través del link que está escrito detrás del router entramos a su web, utilizando la contraseña admin.

- Ya en la web, comprobamos si el modo de operación del router era dinámico o estático.

- También conectamos una computadora a la red y enviamos un ping al router para comprobar si se había conectado correctamente.

- Conectamos el ventilador de una raspberry pi al pin de 5V y al pin de tierra para poder iniciarla.

- Conectamos la raspberry pi al router mediante un cable UTP y nos fijamos su ip en la web.

- Teniendo su ip enviamos un ping desde la computadora.
