Control de versiones: GIT HUB

Durante el desarrollo de este proyecto incluimos el uso de Git Hub, una plataforma que permite subir archivos GIT. GIT, es un controlador de versiones que gestiona los cambios que se realizan sobre los elementos de algun producto o su cofiguración. La siguiente documentación incluye las habilidades adquiridad con GIT HUB:

  • Generación de Token: Cuenta de GitHub> Perfil> Setting> developer settings> Personal Acces token> Classic> Generar nuevo token> generar nombre> generar días (90 dias)> guardar URL.
  • Clonar el repositorio principal: git clone > enlace del repositorio.
  • Entramos a la carpeta:cd> expo-técnica-2024 (nombre del archivo).
  • Generar la rama grupo2: git branch grupo2 (nombre de la rama).
  • Entrar a la rama grupo2: git checkout grupo2.
  • Editar el archivo: code .
  • Ver el estado del archivo: git status.
  • Cambiar el estado del archivo. Confirmar cambios: git add .
  • Comprobar el cambio del archivo: git status.
  • Ver contenido del archivo: git log.
  • Crear Commit: git commit -m “mensaje”.
  • Subir cambios al repositorio: git push origin grupo2 (nombre de la rama).

Luego de aprender correctamente el funcionamiento del GIT, trabajamos con una Raspberry Pi y un router. A continuacion, adjuntamos la documentacion del proyecto.

Dicho proyecto consiste en crear una Red a la cual se conecten una raspberrypi y una computadora (ambos serían los hosts) a un router mediante wi-fi. Para alcanzar esto, debemos asignarles una IP a los hosts y configurar el router. Primero, debemos resetear el router para que su nombre y contraseña se reestablezcan al estado de fábrica. Seguiremos conectándonos, desde la PC a la red del router, aunque no disponga de acceso a internet. Para esto, entraremos a la página del router: • Ingresar en el navegador la dirección Ip de este (192.168.0.1/24), • Ingresar el usuario “admin” y la contraseña “admin”. Así, podremos verificar el estado de la red, el modo de operación (para controlar si la IP es dinámica o estática) y la lista de clientes DHCP. En esta última, veremos que dispositivos están conectados a la red, a la cual se les asignaran las Ip entre 100 y 199 debido a que este es el rango de asignaciones de Ip del router. Desde la Pc, abriremos la consola y ejecutaremos el comando ping (ingresando la ip del router) para testear la conexión de estos. Para realizar la conexión de la raspberrypi, primero debemos armarla. Conectaremos los cables del del ventilador con los pines, el cable positivo al pin de 5V (5 voltios) y el cable negativo al GND (ground/tierra). Enchufamos la raspberrypi a la corriente y conectamos el cable UTP en la raspberrypi y en el router. Así, la raspberry, aparecerá en la lista de clientes DHCP de la página del router, afirmando su conexión a la red. Finalmente, ejecutaremos el comando ping desde la Pc ya conectada para verificar la conexión de la raspberry al router.

PROYECTO RASPBERRY PI

Conexión y acceso al router

Lo primero que hay que hacer es conectar el router a la corriente y luego entrar a su configuración, para esto hay que presionar la tecla WINDOWS + R y en el recuadro escribir “cmd”, una vez aquí, escribir “ipconfig” y buscar la puerta de enlace predeterminada (dirección del router). La dirección que salga aquí debemos escribirla en el navegador, saldrán campos para ingresar una contraseña y usuario, esto depende del router pero normalmente ambos son “admin”.

Verificación de dispositivos conectados.

Una vez aquí, nos vamos al apartado de DHCP y luego Clientes de DHCP, en esta sección aparecerán los dispositivos conectados a la red. Comprobaremos que esté nuestro dispositivo y veremos la dirección IP que se nos ha asignado.

Conexión de la Raspberry Pi al router.

Ahora deberemos conectar la Raspberry Pi a la corriente y luego a alguno de los puertos LAN del router mediante un cable UTP. Después de hacer la conexión comprobaremos si en el apartado de Clientes de DHCP aparece nuestro Raspberry Pi.

Acceso remoto y prueba de conectividad.

Usando acceso remoto a través del protocolo de SSH, entramos a la consola del Raspberry Pi, después de obtener la dirección IP podemos realizar una prueba, enviando un comando "ping" desde la Raspberry Pi hacia el router. Esto se realiza desde la consola escribiendo “ping” seguido de la dirección IP del router.
